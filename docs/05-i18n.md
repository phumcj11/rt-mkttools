# 05 — i18n / Localization

## หลักการ
- **ภาษาเริ่มต้น: ไทย (`th`)** — ทุกข้อความ default เป็นภาษาไทย
- **เตรียมอังกฤษ (`en`)** ไว้สำหรับอนาคต (โครงสร้างพร้อม แต่ยังไม่เปิดใช้เต็ม)
- ไม่ hardcode ข้อความใน UI — ทุกข้อความผ่าน dictionary key

## Frontend (Next.js)
- ใช้ **locale-based routing**: `/[locale]/...`
  - `/th/dashboard` (ค่าเริ่มต้น), `/en/dashboard` (อนาคต)
- Dictionary: `frontend/src/i18n/messages/`
  - `th.json` — ภาษาไทย (ครบทุก key)
  - `en.json` — ภาษาอังกฤษ (เตรียมไว้)
- กลยุทธ์: redirect locale ที่ไม่ระบุไปยัง `DEFAULT_LOCALE=th`
- รูปแบบ key แบบ namespaced เช่น `nav.dashboard`, `auth.login`

ตัวอย่างโครงสร้าง dictionary:
```json
{
  "nav": { "dashboard": "แดชบอร์ด" },
  "auth": { "login": "เข้าสู่ระบบ" }
}
```

## Backend (NestJS)
- Error / notification messages ผ่าน i18n catalog: `backend/src/i18n/`
- เลือกภาษาจาก: header `Accept-Language` หรือ `user.locale` / `tenant.locale`
- ค่าตั้งต้น fallback = `th`

## Database
- คอลัมน์ `locale` ใน `tenants`, `users`, `content_items`, `ai_templates`
- เนื้อหาที่ผู้ใช้สร้าง (เช่นคอนเทนต์ AI) เก็บภาษาไว้ที่ระดับ record

## Environment
```
DEFAULT_LOCALE=th
SUPPORTED_LOCALES=th,en
NEXT_PUBLIC_DEFAULT_LOCALE=th
```

## การเพิ่มภาษาในอนาคต
1. เพิ่มไฟล์ dictionary ใหม่ (เช่น `en.json` ให้ครบทุก key)
2. เพิ่ม locale ใน `SUPPORTED_LOCALES`
3. เพิ่ม catalog ฝั่ง backend
4. ทดสอบ routing `/en/...` และ fallback
